## 一、项目定位与核心叙事框架

项目名称方向（可选）：

- 「未来校园共创计划」 — 强调参与感与共同创造
- 「我的校园2050」 — 时间感 + 个人视角，易产生代入感
- 「校园漫游者」 — 游戏感强，适合非专业公众

核心叙事逻辑：

要让非专业公众愿意参与设计，需要解决一个心理障碍："我不懂设计，我的想法有用吗？"
因此整套文案的核心叙事应该是：

> "你不需要懂设计，你只需要告诉我们你的感受。每一个在这里生活过的人，都是这座校园最好的设计师。"

这句话作为整个项目的贯穿主线，在每个交互节点上持续强化。

## 二、用户旅程与交互流程设计

整个体验分为六个阶段，每个阶段有对应的交互设计和文案策略。

```
Landing → 场景漫游选点 → 输入创意 → 确认设计说明 → 生成与呈现 → 社区沉淀
 (一)        (二)          (三)        (四)           (五)         (六)
```

### 阶段一：进入与吸引（Landing）

**目标：** 30秒内让用户理解"这是什么"并愿意继续

**页面元素：**

- 动态标题动画：校园三维模型缓慢旋转，若干光点在建筑上闪烁，暗示"这里有故事"
- 主标题（大字）：这座校园，还可以更好
- 副标题：告诉AI你的想象，它来帮你把它造出来
- 引导按钮：开始探索 →
- 底部小字说明：无需注册 · 无需设计经验 · 3分钟完成

**关键文案原则：** 去除所有专业词汇（不出现"参数化""公众参与""Agent"等术语），用日常语言建立信任。

### 阶段二：场景漫游与选点（Explore）

**目标：** 让用户在三维校园中自由游览，找到"让自己有感觉的地方"

**3D 场景展示：** 通过 UE Pixel Streaming 将三维校园场景推流到网页浏览器，用户无需安装任何客户端。

**交互设计：**

- 提供漫游模式（自动飞行巡游）和自由探索模式（鼠标/键盘控制视角）两种入口
- 漫游路径经过精心设计，覆盖校园典型场景：入口广场、图书馆前庭、宿舍区庭院、架空层、屋顶等
- 建筑可高亮悬停，显示简短的场景标签，如："午后最热的角落" / "下雨天没有遮挡的路" / "没人去的好地方"——用情绪化语言而非功能性描述来标注空间

**引导文案：**

- 漫游开始前：走进这座校园，找一个让你有感觉的地方——可以是你觉得不够好的，也可以是你想象中它本来的样子
- 悬停提示：点击这里，告诉我们你的想法
- 等待中的空屏引导：还没找到？试试去图书馆后面看看……

**选点完成后：** 系统自动记录位置坐标并截取当前视角截图，作为后续图生图的参考图。

### 阶段三：输入创意（Create）

**目标：** 让用户用自然语言输入非专业的设计想法，消除表达障碍

**交互设计：** 输入界面要做到极度低门槛，分三层递进：

#### 第一层：情绪选择（选填）

用图标+词语让用户先描述"感受"，不是功能：

😤 不够用 / 😌 太单调 / 🌿 想要绿色 / ☀️ 采光不好 / 🧍 太拥挤 / 🔇 太吵了

#### 第二层：自由输入框

占位文字（placeholder）示例要具体、生活化，而不是空白或者"请输入描述"：

- "这里的长椅太少了，下课想坐下来发发呆"
- "希望这个角落能种一棵树，夏天可以乘凉"
- "感觉这条路走起来不安全，晚上没有灯"

输入框上方提示语：用你最自然的语言说就好，哪怕只有一句话

#### 第三层：可选补充（折叠展开）

- 时间偏好：我希望在 [早晨/午后/傍晚/夜晚] 使用这里
- 使用方式：我想在这里 [独处/和朋友/工作/休息/路过]

提交按钮文案：让AI来理解我的想法 →（而不是"提交"或"生成"）

### 阶段四：确认设计说明（Confirm）

**目标：** 让用户看到 AI 如何理解了自己的想法，给予确认或修改的机会，增强参与感和信任感

**交互设计：**

用户提交创意后，AI 将非专业描述翻译为结构化的设计改进说明，展示给用户确认：

**展示内容：**

- 用户原始输入（回显，让用户知道自己被听到了）
- AI 理解的设计方向（用简洁的要点列出）
- 具体的改进方案描述

**示例展示：**

```
你说的：
"这里的长椅太少了，下课想坐下来发发呆"

AI 理解了你的想法：
• 你希望在这里增加停留和休息的空间
• 适合午后使用，个人独处的场景

准备生成的设计方向：
在现有步道旁增设木质长椅组，搭配低矮绿化隔断，
营造半私密的休息角落，保留原有通行功能。
```

**用户操作：**

- 确认生成 → 进入阶段五
- 修改想法 → 返回阶段三重新输入，保留已有内容
- 调整方向 → 直接编辑设计说明文本后确认

**引导文案：**

- 页面标题：这是AI对你想法的理解——看看对不对？
- 确认按钮：就是这个感觉，生成设计 →
- 修改按钮：不太对，我想调整一下

### 阶段五：生成与呈现（Generate）

**目标：** 让等待过程有参与感，让结果有惊喜感和认同感

**等待动效文案**（逐步出现，配合加载动画）：

1. AI正在理解你的想法……
2. 正在分析这个空间的光线与尺度……
3. 正在把你的感受转化为空间语言……
4. 你的校园，正在生长中……

这组文案的目的是让用户感知到"我的语言被认真对待了"，而不是冷冰冰的进度条。

**结果呈现页：**

- 生成图片以动态展示方式出现（渐显 + 轻微缩放），营造"揭晓"感
- 并排展示：原始视角截图 vs 生成后的新场景图，让对比一目了然
- 图片旁边附上一段 AI 的回应文字，用第一人称向用户解释它"理解"了什么：

> "你提到希望这里能有更多遮阴和停留的地方。我保留了原有的开放视野，在东侧加入了一组轻质廊架结构，搭配低矮的种植箱，让这个角落在午后变成可以安静坐下来的地方。"

这段文字至关重要——它把技术生成结果转化为一次对话的回应，让用户感到被理解。

**底部操作：**

- 重新调整想法 → 返回阶段三
- 分享这个设计 → 存入社区，进入阶段六
- 继续探索其他地方 → 返回阶段二

**3D 模型体验（异步）：**

分享设计后，系统会在后台异步将生成的图片转换为 3D 模型，并置入 UE 场景中。用户后续可以在漫游时回到这个位置，体验自己设计的 3D 构筑物。

状态提示：

- 模型转换中，稍后你可以在校园里看到它——
- ✅ 你的设计已经出现在校园里了！去漫游中找找看

### 阶段六：社区沉淀（Gallery）

**目标：** 展示所有参与者的创意，形成集体共创感

**页面概念：** 一张动态的校园地图，所有用户选点以光点标注，密度越高的区域越亮——直觉上呈现出"哪里最需要改变"。

**页面元素：**

- 页面标题：所有人的校园想象
- 副标题：这里的每一个点，都是一个人在这座校园里的感受与期待
- 统计数据展示（增强参与感）：已有 xxx 位访客参与 · 覆盖校园 xx 个区域
- 点击某个光点 → 显示对应用户的原始输入 + AI 设计说明 + 生成结果对比图

**社交互动：**

- 每个设计卡片下方有点赞按钮 ❤️，显示点赞数量
- 点赞无需登录，基于会话记录防止重复点赞
- 热门设计排序：可按"最新"和"最受欢迎"切换查看

**设计详情弹窗内容：**

- 用户原始想法（文字）
- 情绪标签
- AI 回应文字
- 原始场景 vs 生成效果对比图
- 点赞数
- 3D 模型状态（已生成 / 生成中）

## 三、整体文案语气规范

为了保持一致性，制定如下语气原则：

| 维度 | 方向 |
|------|------|
| 人称 | 用"你"而非"用户"，用"我们"而非"系统" |
| 专业词汇 | 全程回避，AI生成→"AI来理解"，参数化→不出现 |
| 情感基调 | 温暖、好奇、有点诗意，但不煽情 |
| 句式 | 短句为主，多用破折号和省略号制造呼吸感 |
| 错误提示 | 鼓励式："再多说一点点，AI会理解得更好" 而非 "输入不能为空" |
| 等待状态 | 叙事式：用分步文案替代冷冰冰的加载条 |

## 四、关键体验设计原则

1. **降低第一步的门槛，而不是最后一步。** 进入越容易，后续参与才越深。首屏不能有任何注册或选项。
2. **让"不专业"变成优势。** 文案要反复传递：普通人的直觉感受，正是设计专业学生忽略的东西。
3. **让用户感到被理解，而不是被执行。** 确认步骤（阶段四）和 AI 回应文字是整个体验最关键的两个环节，需要做到个性化、有温度。
4. **集体感是留存的关键。** Gallery 页的设计要让用户看到"我的点"在整个校园图景中的位置，产生"我参与了一件大事"的归属感。
5. **对比创造冲击力。** 展示结果时始终并排呈现"原场景 vs 新设计"，让改变可视化、可感知。
