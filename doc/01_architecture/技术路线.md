## 前端技术方案

### UE 场景渲染（大团队负责）

校园三维场景由大团队使用 UE 平台搭建，包含：
- 校园建筑模型、地形、植被等基础场景资产
- 漫游路径和摄像机控制逻辑
- 场景中的交互点标记系统

### Web 端展示方案：UE Pixel Streaming

将 UE 场景通过 Pixel Streaming 推流到网页浏览器：

| 项目 | 说明 |
|------|------|
| 技术方案 | UE Pixel Streaming / Pixel Streaming 2 (UE 5.4+) |
| 通信协议 | WebRTC（视频流下行 + 输入上行） |
| 前端集成 | UE 前端 JS SDK 嵌入到网页 iframe 或 div 中 |
| 硬件要求 | 服务端需要 GPU 支持实时渲染 |
| 延迟 | 局域网 < 50ms，公网取决于网络质量 |

**参考资源：**
- [UE Pixel Streaming 官方文档](https://docs.unrealengine.com/5.0/en-US/pixel-streaming-in-unreal-engine/)
- [Pixel Streaming 2 概述](https://docs.unrealengine.com/5.4/en-US/pixel-streaming-2-in-unreal-engine/)

### 非 3D 页面

Landing 页面、创意输入界面、社区 Gallery 等非 3D 页面使用原生 HTML / CSS / JavaScript 开发，通过 FastAPI 提供静态文件服务。3D 漫游区域通过 Pixel Streaming 嵌入。

## 完整数据流程

```
用户操作                  后端处理                      外部服务
────────                ────────                    ──────────

1. 进入Landing页
   │
2. 浏览UE校园场景
   (Pixel Streaming)     ←── UE服务器实时渲染
   │
3. 选中某个位置
   │  (位置坐标 + 当前视角截图)
   │
4. 进入创意输入界面       →  提交用户输入
   (情绪+文字描述)            │
                              ↓
                         5. 交互Agent处理        →  LLM API
                            - 理解非专业描述
                            - 生成设计改进说明
                            - 返回给用户确认
                              │
                         6. 用户确认说明 ←───── 返回确认结果
                              │
                              ↓
                         7. 图生图生成           →  图生图模型 API
                            - 组合：截图 + 设计说明
                            - 生成新场景图
                              │
                              ↓
                         8. 结果展示
                            - 生成图展示
                            - AI 回应文字说明
                              │
                         9. 用户选择
                            ├─ 重新调整 → 回到步骤4
                            ├─ 分享设计 → 存入数据库
                            └─ 继续探索 → 回到步骤2
                              │
                         10. 图转3D（异步）       →  图转3D模型 API
                            - 提取新增/修改部分
                            - 生成3D模型
                            - 置入UE场景
                              │
                              ↓
                         11. 社区展示
                            - 数据库记录
                            - Gallery地图展示
                            - UE漫游中查看缩略图
```

## AI 模型选型

### 交互 Agent（理解用户意图）

| 项目 | 说明 |
|------|------|
| 功能 | 将非专业描述翻译为专业设计语言 |
| 输入 | 用户的情绪标签 + 自由文字描述 + 位置信息 |
| 输出 | 结构化的设计改进说明（面向用户确认） |
| 候选方案 | Claude API / GPT API / 国产大模型 API |
| 要求 | 支持中文理解、长文本生成、指令跟随能力强 |

### 图生图模型

| 项目 | 说明 |
|------|------|
| 功能 | 基于场景截图 + 设计说明生成新的场景图 |
| 输入 | 当前视角截图（参考图） + 设计改进说明（Prompt） |
| 输出 | 新的场景效果图 |
| 候选方案 | Stable Diffusion (ControlNet) / Midjourney API / ComfyUI 自部署 |
| 要求 | 保持原图结构的同时添加/修改指定元素，风格一致性 |

### 图转 3D 模型

| 项目 | 说明 |
|------|------|
| 功能 | 从生成的场景图中提取新增/修改部分，生成 3D 模型 |
| 输入 | 生成图 + 变更区域标注 |
| 输出 | 3D 模型文件（FBX/glTF） |
| 候选方案 | 腾讯混元 3D (Hunyuan3D) / Tripo3D / Meshy |
| 要求 | 能识别并分离新增构筑物，输出可导入 UE 的模型格式 |

## 玩法流程

### 阶段一：场景漫游

场景搭建使用 UE 平台，由大团队共同开发。Web 端通过 Pixel Streaming 进行展示。

### 阶段二：交互 Agent 引导

交互 Agent 负责辅助不了解建筑设计的公众参与设计，将用户的非专业术语翻译为专业设计语言。

流程：
1. 用户在 UE 场景中选点并进入编辑模式
2. Agent 引导用户描述想法（情绪选择 + 自由输入）
3. Agent 理解用户意图，生成详细的设计改进说明
4. **用户确认**改进说明（可修改后重新生成）
5. 确认后将说明 + 当前视角截图发送给图生图模型

### 阶段三：玩家场景社区

- 所有玩家生成的设计记录存入数据库（用户信息、场景图、设计说明、时间等）
- 前端 Gallery 页面以校园地图形式展示所有设计，光点密度反映关注度
- 用户可以给喜欢的设计点赞
- UE 场景中也可以在漫游时查看各位置的玩家设计缩略图（参考地图软件实景模式中的游客上传图片交互逻辑）

### 阶段四：图转 3D 模型

- 将玩家生成的图片发送给图转 3D Agent（使用 Hunyuan3D 等模型）
- 需要识别并提取出修改的部分（如新增的遮阳装置），仅生成变更部分的 3D 模型
- 将新模型置入 UE 场景，让玩家在三维场景中体验自己的设计
- 如果涉及已有构筑物的编辑，识别应替换的部分，用新模型替换旧模型

## API 路由规划

```
# 会话管理
POST   /api/v1/sessions              创建访客会话

# 设计流程
POST   /api/v1/designs/input          提交创意输入 → Agent 处理
POST   /api/v1/designs/confirm        确认设计说明 → 触发图生图
GET    /api/v1/designs/{id}           获取设计详情
GET    /api/v1/designs/{id}/status    查询生成状态（轮询）

# 社区
GET    /api/v1/designs                获取设计列表（支持分页、排序）
GET    /api/v1/designs/map            获取地图光点数据
POST   /api/v1/designs/{id}/like      点赞
DELETE /api/v1/designs/{id}/like      取消点赞

# 统计
GET    /api/v1/stats                  获取参与统计数据
```
