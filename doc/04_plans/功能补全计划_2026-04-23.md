## 五、技术方案建议

### 5.1 用户认证方案

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| OAuth2（清华邮箱） | 体验好、安全 | 需申请第三方登录 | ⭐⭐⭐⭐⭐ |
| JWT 邮箱登录 | 简单可控 | 需自建邮件服务 | ⭐⭐⭐⭐ |
| 简单密码登录 | 最简单 | 安全性较低 | ⭐⭐⭐ |

**推荐**：清华 OAuth2 + 邮箱作为备用

### 5.2 搜索/筛选方案

```sql
-- 按位置筛选
WHERE location_label = '图书馆'

-- 按情绪标签筛选
WHERE emotion_tags @> ARRAY['crowded']

-- 按热度排序
ORDER BY likes_count DESC, created_at DESC
```

前端实现：URL 参数 + API 查询

### 5.3 分享海报方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| html2canvas | 纯前端实现 | 图片质量一般 |
| 服务端生成 | 质量可控 | 需额外依赖 |

**推荐**：html2canvas（快速实现）

## 六、数据库变更

### 6.1 新增表（用户系统）

```sql
-- 用户表
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    nickname VARCHAR(100),
    oauth_provider VARCHAR(50),  -- 'tsinghua', 'email'
    oauth_id VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    last_login_at TIMESTAMP
);

-- 设计与用户关联（修改 designs 表）
ALTER TABLE designs ADD COLUMN user_id INTEGER REFERENCES users(id);
ALTER TABLE designs ADD COLUMN is_public BOOLEAN DEFAULT TRUE;
```

### 6.2 新增表（评论系统，可选）

```sql
CREATE TABLE comments (
    id SERIAL PRIMARY KEY,
    design_id INTEGER REFERENCES designs(id),
    user_id INTEGER REFERENCES users(id),
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
```
