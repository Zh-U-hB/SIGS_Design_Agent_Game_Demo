## 一、技术架构

### 1.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | FastAPI (Python 3.13) | 异步 Web 框架 |
| 包管理 | uv | Python 依赖管理 |
| ORM | SQLAlchemy + asyncpg | 异步 PostgreSQL 访问 |
| 配置 | pydantic-settings | 从 `.env` 读取环境变量 |
| 代码检查 | Ruff | lint + format |
| 测试 | pytest + pytest-asyncio + httpx | 异步测试框架 |
| 前端 | 原生 HTML / CSS / JavaScript | 无框架，静态文件 |
| 前端服务 | live-server (npm) | 开发时热重载 |
| 数据库 | PostgreSQL (云) | 无本地数据库 |

### 1.4 数据库模型

```
User (用户/会话)
├── id: UUID (主键)
├── session_id: String (唯一, 索引) — 访客标识
└── created_at: DateTime

Design (设计记录)
├── id: UUID (主键)
├── user_id: UUID (外键 → User)
├── location_x/y/z: Float? — 选点坐标
├── location_label: String? — 位置名称
├── emotion_tags: JSONB? — 情绪标签数组
├── user_input: Text? — 用户原始描述
├── design_description: Text? — AI 生成的设计说明
├── original_screenshot: String? — 原始场景截图 URL
├── generated_image: String? — 生成图片 URL
├── model_3d_url: String? — 3D 模型 URL
├── ai_response: Text? — AI 原始响应
├── likes_count: Integer (默认 0)
└── created_at: DateTime

Like (点赞记录)
├── id: UUID (主键)
├── user_id: UUID (外键 → User)
├── design_id: UUID (外键 → Design)
├── created_at: DateTime
└── 唯一约束: (user_id, design_id)
```

### 1.5 统一响应格式

```json
{
    "code": 0,
    "message": "success",
    "data": { ... }
}
```

错误码：`0` 成功 | `40001` 参数错误 | `40101` 认证失败 | `40401` 未找到 | `50001` 服务器错误

### 2.2 前端六阶段用户流程（全部接通 API）

```
landing.html ──POST /sessions──→ 获得 sessionId
    │
    ▼

    │
    ▼

    │
    ▼

    │
```

**各页面具体实现：**

| 页面 | API 调用 | 说明 |
|------|---------|------|
| landing | `API.createSession()` | 点击「开始探索」创建会话，复用已有 sessionId |
| explore | `API.createSession()` + 选点存储 | 页面加载确保有 session；选点存储坐标（从 CSS 百分比转换） |
| create | `API.submitDesign()` | 提交完整数据（session_id、坐标、情绪标签、文字），跳转到 confirm |
| confirm | `API.confirmDesign()` | 展示 AI 响应，确认后触发生成流程 |
| generate | `API.getDesignStatus()` 轮询 | 2 秒间隔轮询，最多 60 次；无 designId 时回退模拟模式 |
| gallery | `API.getMapData()` + `API.getStats()` | 并行获取，API 失败时回退演示数据 |

```python
async def translate_to_design_spec(

    location_y: float | None,
```
